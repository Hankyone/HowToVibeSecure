Hardening Basics

  • CSP: Default‑src 'self'; avoid unsafe‑inline.
  • Headers: Strict‑Transport‑Security, X‑Content‑Type‑Options, Referrer‑Policy.
  • Auth: Use short‑lived tokens, rotate secrets.
  • Inputs: Validate on server and client.
  • Dependencies: Pin versions, verify integrity.

Ship Checklist

  1. Threat model and data classification
  2. Secure defaults and headers
  3. Input handling and output encoding
  4. Dependency and secret hygiene
  5. Monitoring and incident playbook
